# knotch — Claude Code plugin

Installable bundle of the skills and hooks that wire a Claude Code
session into a knotch-managed workspace.

## Contents

| Path                                | What                                                          |
|-------------------------------------|---------------------------------------------------------------|
| `.claude-plugin/plugin.json`        | Plugin manifest (name, version, author, license)              |
| `hooks/hooks.json`                  | Authoritative mandatory hook block                            |
| `skills/<name>/SKILL.md`            | `/knotch:query`, `/knotch:mark`, `/knotch:gate`, `/knotch:transition` |
| `knotch-optional-hooks.example.jsonc` | Opt-in hook blocks — copy into `.claude/settings.json` to enable |

The `skills/` directory is generated by `cargo xtask plugin-sync`
from the project root's `.claude/skills/knotch-<name>/` sources;
the prefix is stripped so plugin-mode invocation uses the clean
namespaced name `/knotch:<name>`.

## Requirements

- Claude Code **v2.1.85+** (required for the `if` field on hook handlers).
- `knotch` binary on `$PATH` — `cargo install knotch-cli`.
- Target workspace already initialized with `knotch init`.

## Install

```bash
/plugin install knotch
```

Claude Code merges `hooks/hooks.json` into the active settings file.
Optional hooks stay out of the main settings — see
`knotch-optional-hooks.example.jsonc` for opt-in blocks.

## Standalone alternative

If you're not using the plugin system:

```bash
knotch init --with-hooks --hook-target project
```

The CLI performs the same merge directly against
`.claude/settings.json`. Optional hooks require
`--include-optional`.

## Regenerating

```bash
cargo xtask plugin-sync          # rebuild skills from .claude/skills/
cargo xtask plugin-sync --dry-run # preview changes
```

`plugin-sync` is destructive — the `skills/` directory is a derived
artifact. Hand-edit `.claude/skills/knotch-<name>/SKILL.md` and
re-sync.
